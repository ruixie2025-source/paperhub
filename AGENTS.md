# PaperHub

## Project Goal

Build a paper management system focused on Digital Economy research.

## Tech Stack

- React
- Vite
- TypeScript
- TailwindCSS
- shadcn/ui
- Python
- FastAPI
- SQLite
- SQLAlchemy

## Rules

- Think before coding.
- Explain the implementation plan before modifying code.
- Keep files modular.
- Do not create unnecessary files.
- Write clear comments.
- Use snake_case naming.
- Keep functions short.
- Ask for confirmation before installing dependencies.
- Never delete user files unless explicitly requested.

## Workflow

Before writing code:

1. Understand the task.
2. Explain the plan.
3. Wait for approval.
4. Modify the code.
5. Explain what changed.
